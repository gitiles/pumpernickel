      • setDocuments

        public void setDocuments(Document newSelectedDocument,
                         Document[] newOpenDocuments)
        This sets both the current selected document and the list of open documents at the same time.

        If you call setSelectedDocument(..) and then getOpenDocuments().setAll(..) then one call will trigger listeners before the other call is complete. This could lead to funky results (for example: the selected document may not exist in the list of currently open documents). This method will not fire any listeners until both the selected document and the list of currently open documents has been updated.

        newSelectedDocument -
        newOpenDocuments -

      • tryToCloseWindow

        public void tryToCloseWindow()
        This method is called when the user tries to close the JFrame these controls relate to.

        The default implementation here walks through every document. If it is dirty, the CloseDocumentAction is used to walk the user through saving (or not).

        This method may throw a UserCancelledException, but if it finishes successfully it should call getFrame().setVisible(false) at the end.
