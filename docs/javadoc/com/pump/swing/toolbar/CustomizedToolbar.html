    public class CustomizedToolbar
extends javax.swing.JPanel
    This toolbar mimick's Apple's standard toolbars and their drag-and-drop customization.

    There are certain assumptions made about this component, such as:

    • This will be given lots of horizontal space.
    • Components you try to display in it will be of similar "reasonable" heights.
    • This will be used in a JFrame.
    • You will not need to call setVisible() on components in this toolbar.

    There is room for improvement in this implementation. As of this writing:

    • The "show icon/text" combobox is not yet implemented.
    • Cocoa toolbars elegantly cut off components and have left/right scrolling arrows when the toolbar is not given enough horizontal space: this toolbar currently does not address this problem.
    • Also there are at least 2 TODO notes in this class itself, commenting on aesthetic bugs.

      • CustomizedToolbar

        public CustomizedToolbar(javax.swing.JComponent[] components,
                         java.lang.String[] defaults,
                         java.lang.String toolbarName)
        Creates a new CustomizedToolbar.
        Parameters:
        components - the components that may be in this toolbar.

        Each component must have a unique name; this is how the order of each component is stored between sessions.

        defaults - the default order of the components. This array should contain the names of the components in the previous argument.

        These defaults are used when there are no preferences to indicate how to lay out this toolbar.

        Special/reserved names include:

        • "-": used to indicate a JSeparator.
        • " ": used to indicate a regular gap.
        • "\t": used to indicate a flexible gap.

        You can use as many of these special names as you like, but actual components in the previous argument should only be referenced once.

        toolbarName - a unique name for this type of toolbar. This name serves as a key in the preferences to retrieve the order of the components in this toolbar. Most applications will only have 1 type of customized toolbar. Apple's Mail application has at least 2 toolbars: one in the main window, and one when you compose a new message. So each type of window should use a unique name. If this is null then no preferences will be consulted.

      • displayDialog

        public void displayDialog(int dialogMaxWidth)
        Displays the dialog that lets the user customize this component.

        This covers the underlying JFrame in a transparent sheet so the toolbar can also send/receive drag and drop events appropriately, and to block out the rest of the GUI so the dialog appears modal. (It is not technically modal, because otherwise we would have trouble interacting with the toolbar underneath.)

        Parameters:
        dialogMaxWidth - this is the width used to decide when to wrap rows of components. Depending on the size of your toolbar, and the platform, you may want to change this number.
