    public class JPopover<T extends javax.swing.JComponent>
extends java.lang.Object
    This is similar to a tooltip, except each JPopover is responsible for its own visibility rules. Since tooltips have such a well-defined meaning in Swing this class branches out with a new name for a similar feature.

    This class is inspired by Apple's popover and Microsoft's balloon components.

    By default a JPopover will appear when the mouse hovers over or transfers keyboard focus to the owner component. You can override this behavior by overriding the isStayVisible() method. There can be only one JPopover visible at any time, so when one opens the previous one will close.

        public JPopover(javax.swing.JComponent owner,
                T contents,
                boolean rolloverContents)
        Create a new JPopover.
        Parameters:
        owner - the owner/trigger of this JPopover.
        contents - the contents of this JPopover.
        rolloverContents - if true then this popover stays visible when the mouse moves the mouse over it. If your popover contains components like buttons or textfields that the user needs to interact with then this should be true. If the contents are not interactive (like a label or tooltip) then this should be false.
