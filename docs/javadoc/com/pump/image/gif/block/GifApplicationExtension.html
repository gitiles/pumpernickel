      • GifApplicationExtension

        protected GifApplicationExtension(byte[] header,
                                  byte[] data)
        Creates a GifApplicationExtension. The byte arrays used to create this object are NOT cloned, it's assumed they will not be changed or reused after this object is constructed.
        Parameters:
        header - the 11-byte header of this block. This is the 8-byte application identifier, followed by the 3-byte authentication code.
        data - the application data. In the original GIF file, this is presented as a series of sub-blocks, but this argument should NOT be presented as sub-blocks, it should already be parsed.

      • setApplicationIdentifier

        public void setApplicationIdentifier(java.lang.String s)
        Assigns the application identifier of this block.
        Parameters:
        s - this must be 8 characters. If it is less than 8 characters, then spaces are appended to it to make it 8 characters, if it more than 8 characters, an exception is thrown.

        Also, this must be composed of ASCII-compatible bytes. (That is, characters smaller than an int value of 256.)

      • getApplicationAuthenticationCode

        public byte[] getApplicationAuthenticationCode()
        Returns:
        sequence of three bytes used to authenticate the Application Identifier.

        In practice, this is most often simply a version number. "2.0" is the authenitication code for the popular "NETSCAPE" looping extension.

        The GIF file format specification says:

        An Application program may use an algorithm to compute a binary code that uniquely identifies it as the application owning the Application Extension.

      • setApplicationAuthenticationCode

        public void setApplicationAuthenticationCode(java.lang.String s)
        Assigns the application authentication code for this block.
        Parameters:
        s - a string less than 3 characters in length, that can be converted to ASCII bytes. (If this string is more than 3 characters, or the bytes are not ASCII-friendly, an exception is thrown. The GIF file format pre-dates modern worries about international characters; it has to be ASCII.)

        If this string is less than 3 bytes, then spaces are appended to it to make it exactly 3 bytes.
