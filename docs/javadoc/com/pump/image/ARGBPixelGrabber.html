com.pump.image

Class ARGBPixelGrabber

  • java.lang.Object
    • com.pump.image.ARGBPixelGrabber

  • public class ARGBPixelGrabber
extends java.lang.Object
    This is based conceptually on the java.awt.image.PixelGrabber class, except this is designed to be optimized for several types of BufferedImages. This should both save time and prevent BufferedImage's from becoming un-managed.

    If this is asked to iterate over the image data for a non-supported Image, then this class will defer to a PixelGrabber.

    You'll get the best performance out of this class if you iterate over 1 row of data at a time and recycle the array you're using.

      • getPixels

        public int[] getPixels()
        This returns all the pixels of this image at once.

        This method is provided largely to be consistant with the java.awt.image.PixelGrabber. If it is possible, it is probably more efficient to iterate over the pixel data one row at a time.

        This method returns the entire image in bulk, and does not interact with the "iterator" nature of this object. So calling this method will not affect the state of isDone, or the state of future calls to next(...).

        Calling this method is equivalent to calling:
        getPixels(new int[getWidth()*getHeight()]);

        Returns:
        an array that is getWidth()*getHeight() units long, containing all the ARGB data of this image.

      • next

        public int next(int[] data,
                int arrayOffset,
                int arrayLength)
        Fills the array provided with ARGB pixel data.
        Parameters:
        data - the array to fill. This array can be of arbitrary length, but you may see better performance if you ask for getWidth() many pixels at a time.
        arrayOffset - this is the index to begin writing to in data.

        If the image involved is a BufferedImage of type ARGB_PRE, ARGB or RGB then you will see improved performance if this value is zero.

        arrayLength - this is the amount of data that should be read into data. An exception will be thrown if this method is asked to write more pixel data than will fit in the array.

        If this value is a non-positive number, then an arbitrary number of pixels will be stored in the array.

        Returns:
        the number of pixels written in this array. This will generally be equal to arrayLength, except in 2 cases:
        • When this iterator has finished.
        • When arrayLength is non-positive. In this case this object gets to decide how many pixels to return at a time.

        This will only return 0 if isDone() returns true.
