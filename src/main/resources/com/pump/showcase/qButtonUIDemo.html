This demo features several extensions of the QButtonUI.

Context

Over a decade ago Apple released some customized button UI's in this article. These included different button styles, sizes and segments. But of course this was Mac-specific. (And Aqua-specific at that.) I decided to make my own implementation that would be completely cross-platform.

Why so many options? Believe it or not: there used to be a reason. You should use bevel "with icons, or buttons sized larger than a standard Mac button." Use gradient to "select among options, such as different ways of viewing something." The recessed look was "to indicate a choice in limiting the scope of an operation, such as the buttons at the top of a Finder when searching." The recessed one actually persists (as of 2018) as a Scope Button, but most of the other distinctions have faded away with time. Now gradients in buttons are much less popular. Someday I might redo these (or introduce new ones) to account for this.

But while UI guidelines are trending in a simpler/more flat direction, it's also important to note that the rise of diverse HTML UI's have conditioned users to happily accept variation in their UIs. As long as your UI is well thought-out and relatively uniform: users will generally be OK with it.

Customization / Client Properties

In the demo app all the buttons in a pane share the same instance of one ButtonUI object. The ButtonUI looks at client properties in the button to distinguish the shape, segment position, and other attributes. To set up a button on the bottom-left corner of a grid of buttons, you can call: 

myButton.putClientProperty( HORIZONTAL_POSITION, LEFT );
myButton.putClientProperty( VERTICAL_POSITION, BOTTOM );
(It is assumed if one of these properties is missing that the intended value is ONLY.)

Also check out the ButtonCluster class to help set up your segment client properties automatically.

There is also a client property to define a custom shape: 

myButton.putClientProperty(SHAPE, new Ellipse2D.Float(0,0,100,100));

Apple's original implementation forced buttons to be specific sizes. (This is a very typical Apple thing to do.) The QButtonUI can stretch and fill whatever size you request as you resize the demo window.

ComboBoxUIs

The QButtonUI includes a method to create a matching QComboBoxUI.

Once you get a good-looking toolbar with a specific look and feel: you probably want other components to generally match in colors and size. This helps keep the toolbar/inspector looking unified and professional.
