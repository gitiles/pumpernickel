This demo features three classes:

  • The com.pump.swing.XMLTextComponentHighlighter interacts with the Swing text component to apply special formatting to render XML text.

  • The com.pump.io.xml.XMLParser breaks up text into XML-based tokens. This class is incomplete. It can parse what I call "everyday XML", like what's in this demo. But if you actually study the XML specification: there's a lot of complex witchcraft to support to be a full-fledged XML parser. This meets my needs for now; I'll see if I ever have to scale this up or find another parser. (I did experiment with a couple of existing parsers, but I found they didn't give me the exact positions of each token. If anyone has any leads for an XML parser that I should try, please let me know!)

  • And the faithful com.pump.swing.LineNumberBorder renders the line number to the left of each line.

This is similar to the JavaTextComponentHighlighter.
