The abstract FileIcon class returns a javax.swing.Icon for a file.

This demo includes 2-3 implementations depending on your OS.

AquaFileIcon

This implementation is the default on Mac, and it is only available on Mac.

This uses reflection to invoke the com.apple.laf.AquaIcon.FileIcon class.

FileViewFileIcon

This implementation creates a new JFileChooser and uses its FileView.

This also works on Mac, but it requires the overhead of setting up a new JFileChooser.

FileSystemViewFileIcon

This implementation uses FileSystemView.getFileSystemView() to identify icons.

This does not work on Mac. (It only returns two icons: one for documents and one for folders.)
